AirLLM

A self-hosted, OIDC-governed LLM gateway — one policed entry point between your coding agents and the models.
AI Router LLM · one Go binary · OpenAI- & Anthropic-compatible · Apache-2.0
Get started View on GitHub →

A single Go service exposes OpenAI- and Anthropic-compatible endpoints to internal coding agents, authenticates them by API key, enforces per-key model policy and rolling usage limits, routes to upstream providers with balancing and fallback, and meters tokens and cost. It can scan prompts for secrets and PII (DLP), capture traffic for audit, and feed reviewer-corrected captures back into the detector. A static console, served by the same binary, handles keys, usage, and administration.

# Three ways to run it

All public-clean — secrets are supplied at deploy time, never committed.

Dev / mock

Local development against a mock provider.

make compose-up

Standalone prod

Single-host: persistent, permanent admin, optional auto-HTTPS.

make gen-secrets
make compose-prod-up

Kubernetes

HPA/KEDA autoscaling, secrets by reference, ArgoCD.

helm install airllm deploy/helm/airllm \
  --set image.repository=\
ghcr.io/ipsupport-llc/ipsupport-airllm

# What it does

⇄Dual ingress. OpenAI (/v1/chat/completions, /v1/models) and Anthropic (/v1/messages) — non-streaming and SSE.
⚖Routing & balancing. Alias catalog with priority fallback and per-provider concurrency; explicit provider/model passthrough. All-busy → 429, never a crash.
◷Policy & limits. Per-role allowed models snapshotted onto each key; rolling 5h / 24h / 7d windows in tokens and USD.
🛡DLP. Deterministic (regex + entropy) plus an opt-in BERT-NER sidecar; off/flag/redact/block. Prompts only, by design.
⟳Capture, flywheel & audit. Async redacted-by-default capture, a second-pass that finds misses, a review queue, JSONL export, and an access-logged auditor role.
📈Observability. Prometheus /metrics, per-component latency, in-console sparklines, and Grafana dashboards as code.
⤢Autoscaling. The DLP sidecar pool scales on its own saturation signal — HPA or KEDA (with scale-to-zero) on kubernetes.
🔒Governed. Generic OIDC or local auth; provider credentials sealed with AES-GCM; secrets stay out of the repo.

# Quickstart (local mock)

Run make compose-up — builds and starts Postgres, Redis, and the gateway on 127.0.0.1:8080.
Grab the admin password from the logs, sign in, and create an API key.
Point your agent at the OpenAI or Anthropic endpoint with that key:
# OpenAI-compatible
curl http://127.0.0.1:8080/v1/chat/completions \
  -H "Authorization: Bearer <your-token>" \
  -d '{"model":"mock-gpt","messages":[{"role":"user","content":"hello"}]}'
